# AGD-136 — Improve Navigation Consistency

## Objective

Align header and navigation patterns so movement across admin, staff, patient, and secondary routes feels consistent.

## Navigation Adjustments

| Area | Issue Addressed | Change Applied | Result |
| --- | --- | --- | --- |
| Shared navigation chrome | Headers, profile chips, drawers, and bottom navigation used separate styling and spacing rules in each module. | Added a reusable navigation widget set for branded headers, user chips, drawer items, and mobile bottom-nav items. | Navigation now follows one visual language across modules. |
| Staff and patient dashboards | App bars and drawers duplicated the same concepts with mismatched order, spacing, and selected-state behavior. | Migrated both dashboards to the shared navigation chrome and aligned drawer ordering with primary destinations shown in bottom navigation. | Mobile navigation behaves more predictably and looks consistent between staff and patient flows. |
| Notifications and recycle bin | Secondary routes used simpler headers that did not match dashboard navigation styling. | Updated both screens to use the same shared header treatment with the standard accent divider. | Navigation remains visually consistent when drilling into sub-pages. |
| Admin layout | The admin shell had its own sidebar and top bar styling, and dashboard card navigation still depended on visible card labels. | Refined the sidebar/header styling to match the shared navigation palette, made the profile chip navigate to profile, and separated the admin staff card route from its display title. | Admin navigation now feels aligned with the rest of the app and route behavior is more reliable. |
