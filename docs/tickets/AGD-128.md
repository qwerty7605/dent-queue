# AGD-128 — Cache Notification List Retrieval

## Objective

Improve notification loading speed while keeping new-notification and read/unread state changes accurate.

## Fix Applied

- Kept backend notification list and unread-count retrieval cached per user namespace.
- Explicitly flushed notification caches after bulk `mark all as read` updates, which previously bypassed model events.
- Deferred notification cache invalidation until `afterCommit` for notification model events so newly created notifications cannot race with transaction-bound writes.
- Switched Flutter notification cache keys from role-only to user-aware keys derived from the authenticated user id.
- Preserved mutation invalidation on read actions so refreshed notification lists and unread counts stay correct.
