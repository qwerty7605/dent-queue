# AGD-132 — Improve Table Layout

## Objective

Improve readability and visual consistency across the admin data tables.

## UI Adjustments

| Area | Issue Addressed | Change Applied | Result |
| --- | --- | --- | --- |
| Shared table shell | Admin tables used different row heights, header treatments, borders, and spacing. | Added a reusable `AdminDataTable` widget with consistent header styling, row sizing, borders, striped rows, and horizontal/vertical scroll handling. | Table spacing and header presentation now match across admin screens. |
| Patient accounts table | The table relied on near-default `DataTable` sizing and loosely aligned cells. | Applied the shared table shell and introduced fixed-width columns for number, patient, gender, contact, and action cells. | Rows are easier to scan and action controls line up predictably. |
| Staff accounts table | Column spacing, row layout, and typography differed from the other admin tables. | Migrated the table to the shared shell while preserving the two-line account cell and aligning role, contact, and action columns. | Headers and row spacing now read consistently with the rest of the admin UI. |
| Master list and reports tables | Header clarity and status column alignment varied, especially around compact badges and long values. | Standardized header styling, widened status columns, and constrained key text columns for patient, service, contact, and queue data. | Columns are more clearly separated and status badges no longer crowd adjacent cells. |
