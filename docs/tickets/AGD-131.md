# AGD-131 — Refine Dashboard Card Layout

## Objective

Improve visual consistency of dashboard cards across admin, staff, and patient dashboards.

## UI Adjustments

| Area | Issue Addressed | Change Applied | Result |
| --- | --- | --- | --- |
| Shared dashboard cards | Each dashboard used a different card radius, spacing, and typography scale. | Added a reusable `DashboardStatCard` widget with standardized padding, border radius, icon treatment, and text styling hooks. | Cards now share one visual structure across dashboard screens. |
| Staff dashboard | Summary cards used a separate compact layout with different spacing and hard-coded text sizes. | Replaced the custom summary tiles with the shared card widget and moved to a responsive fixed-height grid. | Staff cards now align with the same spacing and proportions used elsewhere. |
| Patient dashboard | Appointment status cards had their own layout and only supported a fixed two-column grid. | Reworked the status section to use the shared card widget with responsive one, two, and four-column breakpoints. | Card spacing stays consistent from narrow to wide screens. |
| Admin dashboard | The dashboard used oversized wrap-based cards with inconsistent sizing and routing tied to display labels. | Rebuilt the overview cards as a responsive grid using the shared widget and separated route keys from visible titles. | Admin cards now align cleanly with the rest of the dashboards and the staff card navigates correctly. |
