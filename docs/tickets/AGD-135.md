# AGD-135 — Improve Modal/Dialog UI

## Objective

Make dialogs easier to scan and act on by standardizing spacing, action placement, and close/focus treatment across the main modal flows.

## Fix Applied

- Added a shared `AppDialogScaffold` for custom dialogs so modal padding, radius, close affordances, scroll handling, and footer action placement are consistent.
- Migrated the patient/staff appointment booking dialogs, appointment detail dialogs, edit profile dialog, appointment success dialog, admin notifications dialog, and patient cancel-confirmation dialog to the shared shell.
- Added a shared `AppAlertDialog` plus base `DialogTheme` styling so confirmation and export prompts inherit the same spacing, typography, and button treatment without per-screen overrides.
- Kept primary actions anchored in a dedicated footer area, with destructive and secondary actions clearly separated for better visibility.
